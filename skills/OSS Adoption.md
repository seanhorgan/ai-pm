# OSS Adoption Assessment

## When to use
Activate when assessing the adoption, health, and traction of an open-source project managed on GitHub. This is critical for evaluating dependencies, potential partners, or competitive positioning.

## Rubric — Four Key Metrics

Assess the project across four dimensions, looking at both **absolute totals** and **90-day velocity**:

1. **Stars (Awareness & Interest)**
   - **Total:** Cumulative interest over the project's lifetime.
   - **Last 90 Days:** Recent growth in interest.
   - *Look for:* A high ratio of recent stars to total stars indicates a trending project.

2. **Forks (Developer Engagement)**
   - **Total:** Number of times developers have copied the repo for their own use or contribution.
   - **Last 90 Days:** Recent forks.
   - *Look for:* High fork velocity suggests active experimentation or contribution attempts.

3. **Contributors (Community Health)**
   - **Total:** Total number of unique contributors to the project.
   - **Last 90 Days:** Number of active contributors in the last quarter.
   - *Look for:* A project with many total contributors but few recent ones may be in maintenance mode or declining.

4. **Commits (Development Velocity)**
   - **Total:** Historical volume of changes.
   - **Last 90 Days:** Recent commit volume.
   - *Look for:* High recent commit volume shows active development and maintenance.

## Comparative Analysis

The assessment must include comparison against peers:
- **Input:** The skill accepts a target repo and optionally a list of comparison repos.
- **Proposing Comparables:** If no comparables are provided, propose 2-3 relevant industry peers based on the project's domain and scale.

## Actions

1. **Data Collection:** Retrieve the total and last 90 days counts for Stars, Forks, Contributors, and Commits for the target repo and comparables.
2. **Synthesize Scale:** Classify the project's scale (Small, Medium, Large) based on absolute metrics.
3. **Determine Trajectory:** Compare 90-day metrics against totals to determine if the project is **Growing**, **Stable**, or **Declining**.
4. **Comparative Report:** Present the data in a clear table and provide a summary assessment of adoption health relative to the comparables.
